# 🚀 3. What I Would Build In Week 2

If I had another week, I would focus on:
- production readiness,
- data quality,
- and scalability.

The first improvement would be moving pricing data into a structured config system or admin-managed table so pricing updates become easier over time.

I would also improve benchmarking by comparing anonymized reports to generate:
- spend-per-developer insights,
- peer comparisons,
- and optimization benchmarks.

Additional improvements would include:
- stronger API rate limiting,
- honeypot abuse protection,
- verified email sending domains,
- better Open Graph previews,
- and improved sharing flows.

I would also refine:
- mobile responsiveness,
- accessibility,
- and analytics instrumentation.

# 🏁 Final Reflection

This assignment reinforced that building a strong product involves much more than writing code.

The most important parts were:
- making realistic trade-offs,
- designing trustworthy experiences,
- handling edge cases,
- and communicating decisions clearly.

The project evolved significantly during implementation:
- from a simple audit calculator
- into a more complete SaaS-style product experience with:
  <img width="645" height="82" alt="image" src="https://github.com/user-attachments/assets/dd1f199f-4397-4efb-9a7f-263b7cc72ffa" />


The biggest takeaway was understanding how engineering, product thinking, and usability all connect when building something intended for real users instead of only completing a coding exercise.
